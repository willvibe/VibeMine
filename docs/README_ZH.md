# VibeMine 智能 AutoML 平台

**AutoML Intelligence Platform — From Data to Best Model, Automatically**

---

## 一、平台概述

VibeMine 是一款端到端的智能自动化机器学习平台，基于 PyCaret 3.x 构建，提供从数据上传、特征工程、模型训练、性能评估到模型导出的全流程自动化服务。

用户无需编写一行代码，只需上传 CSV 数据文件，平台即可自动完成数据预处理、算法选择、超参数调优、模型集成，并输出最优模型的完整评估报告。

### 核心特性

- 🚀 **零代码 AutoML**：上传数据 → 配置 → 一键训练 → 获得最优模型
- 🧠 **AI 智能解读**：Google Gemini 驱动的模型评估与错误样本分析
- 🌍 **中英文切换**：完整 i18n 国际化支持
- 📊 **多维可视化**：柱状图、雷达图、SHAP 图、特征重要性图
- 🔧 **6 大高级选项**：异常检测、缺失值填充、分层CV、调优、集成、SMOTE
- 📦 **一键导出**：标准 PyCaret .pkl 模型文件 + Python 部署代码

## 四、数据预处理阶段

### 4.1 异常值检测与处理

**实现方式：** Isolation Forest（孤立森林）算法

- 在数据进入训练流程前，自动检测并移除异常值样本
- 阈值设定为 0.05（5%），符合工业标准
- 异常值会影响模型学习真实模式，移除后模型预测更稳定
- **默认启用**，可通过配置页关闭

```
Isolation Forest 原理：通过随机切分数据空间，异常点更容易被"孤立"（fewer splits needed），
因此其平均路径长度更短，天然适合大规模异常检测
```

### 4.2 高级缺失值填充

**实现方式：** PyCaret 内置稳定插补策略

- **数值列**：缺失值直接丢弃（`numeric_imputation='drop'`），避免插补误差传播
- **类别列**：缺失值使用众数（最频繁值）填充
- **为什么不用 Iterative Imputer：** 迭代插补（KNN/LightGBM）在某些数据分布上会累积误差，导致逻辑回归等线性模型预测失效（AUC 计算异常）。VibeMine 选用 PyCaret 内置的稳定插补策略，兼容所有模型
- **默认启用**，可通过配置页调整

### 4.3 分层交叉验证

**实现方式：** Stratified K-Fold

- **分类任务**：3-Fold 分层交叉验证，保持每折中各类别比例与整体分布一致
  - 避免某折因类别不均衡导致评估结果失真
- **回归任务**：3-Fold 随机交叉验证 + 随机打乱
- **为什么重要：** 单次训练/测试划分容易因数据分布不均产生偏差，分层交叉验证通过多次划分取平均，确保评估结果真实可靠
- **默认启用**

## 五、模型训练与优化阶段

### 5.1 多算法并行竞速

VibeMine 同时训练最多 15 种算法，覆盖分类、回归、聚类三大任务类型：

**分类算法（14 种）：**
| 算法 | 简称 | 适用场景 |
|------|------|----------|
| Logistic Regression | LR | 基线模型、概率输出 |
| Random Forest | RF | 通用、鲁棒 |
| Gradient Boosting Classifier | GBC | 高精度、结构化数据 |
| Extra Trees | ET | 高维数据、抗过拟合 |
| XGBoost | XGB | 大规模数据、竞赛首选 |
| Light GBM | LightGBM | 超大数据、实时预测 |
| Decision Tree | DT | 可解释性要求高 |
| K Neighbors | KNN | 非线性、小数据集 |
| Ada Boost | Ada | 弱分类器集成 |
| Linear Discriminant Analysis | LDA | 降维 + 分类 |
| Naive Bayes | NB | 文本、概率推断 |
| SVM (Linear) | SVM | 高维线性可分 |
| Ridge Classifier | Ridge | 正则化线性分类 |
| QDA | QDA | 非线性判别分析 |

**回归算法（10 种）：** 线性回归、Ridge、Lasso、RF、ET、GBR、XGBoost、LightGBM、KNN、SVM

**聚类算法（5 种）：** K-Means、层次聚类、Mean Shift、DBSCAN、Affinity Propagation

### 5.2 贝叶斯超参数调优（Bayesian Optimization）

**实现方式：** Optuna + PyCaret 集成

- 每个模型使用贝叶斯搜索寻找最优超参数配置
- **搜索空间：** 树深度、叶子数、学习率、正则化系数等核心超参数
- **优化目标：**
  - 分类任务 → **Accuracy**（准确率）
  - 回归任务 → **R2**（决定系数）
- **为什么用贝叶斯：** 相比随机搜索和网格搜索，贝叶斯优化利用历史结果引导搜索方向，在有限预算内找到更优解
- **默认启用**

### 5.3 模型集成（Ensemble）

**实现方式：** Bagging + Boosting 双策略

| 策略 | 适用模型 | 原理 |
|------|----------|------|
| **Bagging** | RF、ET | 对同一算法训练多个子模型，结果投票/平均，降低方差 |
| **Boosting** | GBC、AdaBoost、XGBoost、LightGBM | 顺序训练子模型，每轮修正上轮错误，降低偏差 |

- 集成模型通常比任何单一模型表现更稳定、更准确
- VibeMine 自动判断任务类型选择合适的集成策略
- **默认启用**

### 5.4 类别不平衡处理（SMOTE）

**实现方式：** SMOTE（Synthetic Minority Over-sampling Technique）

- 当数据中某一类别占比超过 1:9 时自动检测并提示用户
- 用户可手动开启 SMOTE，通过对少数类进行过采样来平衡训练数据
- **解决什么问题：** 不平衡数据会导致模型偏向多数类，少数类的召回率极低；SMOTE 生成少数类的合成样本，使模型学习更均衡
- **仅分类任务可用**

## 六、特征重要性分析

### 6.1 特征重要性排名

训练完成后，平台自动提取最优模型的 Top 15 重要特征：

- **树模型**（RF、ET、XGBoost 等）：使用 `feature_importances_` 属性
- **线性模型**（LR、Ridge 等）：使用 `coef_` 属性的绝对值
- 以水平柱状图呈现，紫色深浅表示重要性递减
- 帮助用户理解哪些特征对预测结果影响最大

### 6.2 SHAP 特征影响分析（分类任务）

**实现方式：** SHAP（SHapley Additive exPlanations）

- 针对最优分类模型生成 SHAP 图表
- 展示每个特征对单个预测结果的正向/负向影响
- 可展开/收起，方便对比分析
- **为什么重要：** SHAP 从博弈论角度量化每个特征对预测的贡献，比传统重要性更精细

## 七、多维评估体系

### 7.1 评估指标

平台根据任务类型自动计算以下指标：

**分类任务：**
| 指标 | 说明 | 方向 |
|------|------|------|
| Accuracy | 准确率 | ↑ 越高越好 |
| AUC | ROC 曲线下面积 | ↑ 越高越好 |
| Recall | 召回率 | ↑ 越高越好 |
| Precision | 精确率 | ↑ 越高越好 |
| F1 | 精确率与召回率的调和平均 | ↑ 越高越好 |

**回归任务：**
| 指标 | 说明 | 方向 |
|------|------|------|
| R² | 决定系数 | ↑ 越高越好 |
| RMSE | 均方根误差 | ↓ 越低越好 |
| MAE | 平均绝对误差 | ↓ 越低越好 |

**聚类任务：**
| 指标 | 说明 | 方向 |
|------|------|------|
| Silhouette | 轮廓系数 | ↑ 越高越好 |
| Calinski-Harabasz | CH 指数 | ↑ 越高越好 |
| Davies-Bouldin | DB 指数 | ↓ 越低越好 |

### 7.2 性能对比柱状图

- 支持**多指标同时选择**对比分析
- 当选择正负向混合指标时，自动启用**双 Y 轴**（左轴↑越高越好 / 右轴↓越低越好）
- 每根柱子内显示精确数值（3位小数）
- 柱子颜色与图例颜色完全一致，最优模型使用深紫色渐变高亮
- 图表支持直接导出为 PNG

### 7.3 多指标雷达图

- 最多同时展示 6 个维度的指标对比
- 每个维度的刻度基于该指标在所有模型中的**实际数据范围**动态调整（加 15% padding）
- 最优模型使用实线 + 高亮色，其他模型使用虚线
- 支持直接导出为 PNG

### 7.4 详细指标表

- 展示所有模型的完整评估结果
- 数值统一保留 3 位小数
- 最优模型行自动高亮
- 支持导出为 CSV（含 BOM 头，Excel 打开无乱码）

### 7.5 预测错误样本分析

- **100% 保留**所有预测错误的样本（不限制条数）
- 真实值（红色）与预测值（橙色）区分显示
- 便于定位模型的具体弱点

## 八、AI 智能解读

### 8.1 模型性能解读

由 Google Gemini 2.0 Flash 模型生成：

- 分析各模型指标的差异，指出综合表现最优的模型
- 解读最优模型在准确率与泛化能力上的平衡点
- 提示潜在的过拟合风险

### 8.2 错误样本分析

由 Google Gemini 2.0 Flash 模型生成：

- 分析错误样本的分布特点（是否集中在某类样本）
- 判断是否存在特征缺失或异常值导致的误判
- 给出 2-3 条具体的优化建议（增加样本量、特征工程、调整阈值等）

> **按需调用**：AI 分析仅在用户点击按钮时才会执行，不阻塞训练主流程。API Key 仅保存在用户本地浏览器中。

## 九、模型导出与部署

训练完成后，平台导出标准的 PyCaret 模型文件（.pkl），附带 Python 使用代码示例：

```python
from pycaret.classification import load_model, predict_model

# 加载模型
model = load_model('vibemine_model_abc123')

# 对新数据进行预测
predictions = predict_model(model, data=new_dataframe)
```

### 模型文件管理

- 训练完成后，CSV 数据文件自动清理，模型文件保留供下载
- Session 过期后（默认 1 小时），模型文件自动清理
- 训练失败时，所有临时文件立即清理
